# Как протестировать оверлей на Twitch

## 1. Backend должен быть в сети

Репозиторий уже на GitHub. Разверните его, например, на **Vercel**:

- Зайдите на [vercel.com](https://vercel.com), привяжите аккаунт GitHub.
- **Import** → выберите репозиторий `crossout-twitch-backend`.
- Деплой без доп. настроек (Next.js определится сам).
- Запомните URL вида `https://crossout-twitch-backend-xxx.vercel.app`.

**Хранение сессий на Vercel (обязательно для продакшена):**  
Сессии хранятся в Vercel KV (Redis), иначе после деплоя или холодного старта оверлей будет показывать «нет активной сессии». В проекте Vercel: **Storage** → **Create Database** → выберите **KV** (или Upstash Redis из маркетплейса) → создайте и **подключите к проекту**. Переменные `KV_REST_API_URL` и `KV_REST_API_TOKEN` подставятся автоматически.

Либо запускайте локально: `npm run dev` — тогда URL будет `http://localhost:3000` (без KV данные живут только в памяти).

---

## 2. Настройка программы prser

В программе (настройки / Twitch):

- **Backend URL** — URL вашего backend (Vercel или `http://localhost:3000`).
- **Twitch Channel ID** — логин канала **или** числовой ID канала (должен совпадать с тем, что подставите в оверлей).
- Сохраните настройки.

Запустите программу, выберите лог и игрока. Программа будет отправлять данные на backend по этому URL.

---

## 3. Быстрый тест в браузере (без расширения Twitch)

1. Убедитесь, что prser запущена и лог выбран (данные уходят на backend).
2. Откройте в браузере:
   ```
   https://ВАШ_BACKEND_URL/overlay?channel=ВАШ_ЛОГИН_ИЛИ_ID
   ```
   Например: `https://crossout-twitch-backend-xxx.vercel.app/overlay?channel=iexpo`  
   Значение `channel` должно совпадать с полем **Twitch Channel ID** в настройках prser.
3. Должны отобразиться сводка, история и рейтинг.

Так можно проверить логику и данные без Twitch и OBS.

---

## 4. Тест поверх стрима (OBS / любая программа для стрима)

1. В OBS добавьте источник **Browser** (Браузер).
2. В поле **URL** укажите:
   ```
   https://ВАШ_BACKEND_URL/overlay?channel=ВАШ_ЛОГИН
   ```
3. Задайте размер (например, 1920×1080 или под размер сцены).
4. Разместите источник поверх или рядом с игрой и запустите трансляцию.

Так вы увидите оверлей так же, как зрители, без создания расширения Twitch.

---

## 5. Оверлей как расширение Twitch (для зрителей)

Чтобы оверлей показывался **в трансляции** у всех зрителей как расширение канала.

### Важно: ID канала — числовой

В режиме расширения Twitch передаёт оверлею **числовой** ID канала (например `12345678`), а не логин. В программе prser в поле **Twitch Channel ID** нужно указать именно этот числовой ID.

**Как узнать свой числовой ID:**
- Зайдите на [https://www.twitch.tv/ВАШ_ЛОГИН](https://www.twitch.tv) и откройте исходный код страницы (Ctrl+U). Поиск по `"channelId"` или `"broadcaster_id"` — рядом будет число.
- Или используйте API: `https://api.twitch.tv/helix/users?login=ВАШ_ЛОГИН` (нужен Client-ID). В ответе поле `id` — это числовой ID.

### Шаги по созданию расширения

1. **Аккаунт Twitch**  
   Включите двухфакторную аутентификацию (2FA) в настройках аккаунта — без этого расширения создавать нельзя.

2. **Консоль разработчика**  
   Откройте [Twitch Developer Console](https://dev.twitch.tv/console) → войдите → раздел **Extensions**.

3. **Создание расширения**  
   Нажмите **Create Extension** (или «Создать расширение»).

4. **Тип и название**  
   - **Extension name:** например `Crossout Session Overlay`.  
   - **Extension type:** выберите **Video Overlay** или **Видео: на весь экран** (Video - Fullscreen).  
   - Нажмите **Create**.

5. **Версия и ссылки**  
   Перейдите в **Version** / **Create Version** (или «Добавить версию»).  
   Заполните:
   - **Проверка базового URI (testing_base_uri):** `https://ВАШ_БЭКЕНД_URL/` (со слэшем в конце).  
   - Для **Video Overlay** (компонент): **Video - Component Viewer Path** — укажите `overlay` (без .html).  
   - Для **Видео: на весь экран** (Video - Fullscreen): **Видео: путь полноэкранного режима** — укажите `video_overlay.html`. Backend отдаёт тот же оверлей по этому пути.  
   - Или в старом формате **Viewer URL:** `https://ВАШ_БЭКЕНД_URL/overlay` — без `?channel=...`, канал Twitch подставит сам.  
   - При необходимости **Summary** и **Description** (кратко: оверлей статистики Crossout для стримера).

6. **Сохранение и статус**  
   Сохраните версию. Статус станет **Ready for Testing** (или «Готово к тестированию»). Для теста на своём канале модерация не обязательна.

7. **Установка на канал**  
   - Откройте страницу **вашего канала** на Twitch (как стример).  
   - Перейдите в **Расширения** (под плеером или в настройках канала).  
   - Найдите своё расширение в списке и нажмите **Установить** / **Install**.  
   - Включите его в слоте **Video Overlay** (оверлей поверх видео).

8. **Проверка в эфире**  
   Запустите трансляцию. Оверлей должен подхватить числовой ID канала и запросить данные с backend. Убедитесь, что в prser указан тот же **числовой** Channel ID и программа отправляет данные на тот же Backend URL.

### Как сделать окно расширения больше

Размер окна оверлея задаётся **на стороне Twitch**, не в коде.

1. **Через настройки расширения на канале**  
   Зайдите на страницу **вашего канала** на Twitch → **Расширения** (под плеером) → найдите своё расширение в слоте **Video Overlay** → нажмите **Настроить** / **Manage** (или шестерёнку). В настройках слота иногда можно выбрать размер области (если Twitch отображает такую опцию для вашего типа расширения).

2. **Через консоль разработчика**  
   [Twitch Developer Console](https://dev.twitch.tv/console) → **Extensions** → ваше расширение → вкладка **Version** или **Viewer**. Если в форме есть поля **Width** / **Height** или **Dimensions** для Video Overlay — задайте нужные (например, 600×500 или 700×600). Сохраните и создайте новую версию, затем снова установите расширение на канал.

3. **Через OBS (без расширения)**  
   Если нужно полностью контролировать размер и положение: в OBS добавьте источник **Браузер** с URL оверлея (`https://ВАШ_BACKEND/overlay?channel=ВАШ_ID`), задайте ширину и высоту (например, 500×600). Тогда размер окна задаёте только вы, но оверлей видят все, кто смотрит ваш сцену в OBS (т.е. все зрители стрима).

---

## Итог

- **Только проверить данные:** п. 1–3 (backend + prser + браузер с `?channel=...`).
- **Проверить поверх стрима у себя:** п. 4 (OBS + Browser с тем же URL).
- **Полноценно на Twitch для зрителей:** п. 5 (оверлей как расширение).

**Если оверлей показывает «Для этого канала нет активной сессии»:**  
Проверьте, что в prser указан тот же Channel ID (для расширения — **числовой** ID), что backend доступен по указанному URL и что программа уже отправляла данные (выбран лог и игрок).
